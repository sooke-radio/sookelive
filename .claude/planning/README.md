# Planning notes

Markdown planning notes for new features and ongoing maintenance of the Sooke.live site.

Conventions:

- One file per feature or theme, kebab-case names (e.g. `mixcloud-embed.md`, `show-archives.md`).
- Start each note with a short **Goal** section, then **Approach**, **Open questions**, and **Status**.
- `cleanup-suggestions.md` is a running list of code cruft to fix opportunistically — when a feature touches one of those areas, fold the cleanup into that branch and check it off.
- Delete or archive notes once the feature ships and anything durable has moved into `CLAUDE.md` or the README.
